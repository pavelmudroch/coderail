Check `examples` directory for specific language patterns and best practices.

## Goals
- write clear, type-safe code
- keep APIs predictable
- use types to reduce invalid states

## Rules
- prefer explicit, readable types
- avoid unnecessary generic complexity
- keep runtime behavior aligned with types
- use discriminated unions where helpful
- avoid `any`; use `unknown` for untyped values
